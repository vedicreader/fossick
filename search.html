search

web search for AI agents — text, images, news & videos via ddgs, plus direct Google via a stealth browser

fossick searches the web through ddgs, a metasearch library that aggregates many backends (bing, brave, duckduckgo, startpage, mojeek, yandex, wikipedia, …) with no API key and no Docker. search() returns text results; images(), news() and videos() return the corresponding media.

Direct Google is soft-blocked for many IPs (plain HTTP gets a JavaScript-only shell), so google() uses scrapling’s stealth browser to get real Google ranking when you specifically need it. Every result is a plain dict, and all functions share a local TTL cache so repeated queries return instantly.

Google (stealth browser)

Direct Google is soft-blocked for many IPs — plain HTTP (curl/requests, and ddgs’s own google backend) gets a JavaScript-only shell with zero parseable results, regardless of consent cookies or UA. google() sidesteps this with scrapling’s stealth browser (fetch(..., stealthy=True)): it executes the page JS and evades bot detection, so it gets a real SERP. It’s slower (a browser launches per query), so it’s an opt-in source for when you need Google’s own ranking — results are TTL-cached to amortise the cost.
